Understanding File Path

Whenever you want to enter image in your website you simply copy its address and paste it in src.

But what if that image is in your sydtem itself.

In this case we have to introduce file path.

There are two types of file path:

  1. Relative path:

    In relative path you need to show the way from the point where you are.

    Suppose you have four folders and files inside them are in the following way:

    • root

      • lion.png
      • cat.png

      • Alpha

        • dog.png
        • index.html

        • Beta

          • home.html
          • cow.png

          • Photos

            • First.png
            • Second.png

    2. Suppose you are writing code in index.html and you want a picture of dog so you simply write src="dog.png or ./dog.png"

    If you want the picture of cow you should write src="Beta/cow.png or ./Beta/cow.png"

    For second picture write src="Beta/Photos/second.png"

    For cat picture write src="../cat.png"

    If you are writing code in home.html for cat picture write src="../../cat.png"

    In relative path also when it is trying to enter in your system then also browser did'nt support relative path.

  2. Absolute path:

    In absolute path you need to show the way from starting to end.

    Whenever you request for any website and it follows the absolute path to show result in your computer then it start from root to the place where you want to open it.

    So it is dangerous for your computer because the website has access to reach main storage of your computer because in absolute path it starts from zero that's why browser did'nt support absolute path and did'nt show result.

Why use Boilerplate code?

  1. Because it is recommended method.
  2. Also when you run HTML code without using Boilerplate code then it shows result without any error and according to your expectation.
  3. But when you have to use CSS with HTML file then it can show unexpected error that's why we should use Boilerplate code.

Understanding Boilerplate code

  • Doctype html means we are using 5th version of HTML.
  • lang="" showing that most of the content is in which language.
  • Inside html there are two parts head and body
    1. head
      • head is something that is not display in website.
      • head tells meta data.meta data means data about data.
      • Inside head title is used so that you can understand what website is about by reading it from above tab.
      • meta name="viewport" is used so that you get option to open website in desktop-layout.
      • content="width=device-width is used to set width of content according to your device.
      • UTF-8 is a dictionary which contain all letters corresponding to their binary number.
    2. Body

      It is the content which will display in website.

Div Tag

It is like container which contain different tag and whatever same property we want to apply in different tag,instead of writing same property many times in different tag we simply write it in div and it applies to every tag inside it.
